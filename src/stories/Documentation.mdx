# WCDS

## Web Component Design System

A framework-agnostic design system built with Web Components. Use it with React, Vue, Svelte, Angular, Elm — or plain HTML.

---

## Philosophy

**One library, every framework.** Traditional component libraries lock you into a specific framework. WCDS uses native Web Components, which work everywhere — no adapters, no wrappers, no compromises.

**Design tokens as the source of truth.** Every color, spacing value, and radius is defined as a token. Change once, update everywhere.

**Modern by default.** OKLCH colors for perceptually uniform palettes. CSS custom properties for runtime theming. ES modules for tree-shaking.

## Design Tokens

Tokens are the foundation of WCDS. They live in `src/tokens/` and are organized by purpose:

```
src/tokens/
├── core/           # Primitives
│   ├── color.tokens.json      # Color palettes (primary, secondary, neutral)
│   ├── size.tokens.json       # Spacing scale (0-10)
│   ├── radius.tokens.json     # Border radii
│   └── typography.tokens.json # Font families, weights, sizes
├── semantic/       # Purpose-driven aliases
│   ├── background.tokens.json
│   ├── border.tokens.json
│   ├── spacing.tokens.json
│   └── text.tokens.json
├── component/      # Component-specific tokens
│   ├── button.tokens.json
│   ├── input.tokens.json
│   └── ...
└── generated/      # Auto-generated (don't edit)
    ├── design-tokens.css
    ├── design-tokens.js
    └── design-tokens.d.ts
```

### Regenerating Tokens

After editing any `.tokens.json` file:

```bash
pnpm prepare
```

This runs Style Dictionary to generate CSS variables, JavaScript exports, and TypeScript types.

## Usage

### Installation

```bash
npm install wcds
```

### Setup

```js
// Import components and styles
import 'wcds'
import 'wcds/style'
```

Add the font to your HTML:

```html
<link href="https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@400;500;600;700&display=swap" rel="stylesheet">
```

### Using Components

```html
<wcds-button variant="primary">Click me</wcds-button>
<wcds-input placeholder="Enter text..." />
<wcds-badge variant="success">Active</wcds-badge>
<wcds-card>Card content</wcds-card>
<wcds-avatar name="John Doe" />
```

## Theming

All design tokens are exposed as CSS custom properties. Override them to theme your app:

```css
:root {
  --wcds-color-primary-500: oklch(65% 0.28 200); /* Change primary to teal */
  --wcds-font-family-sans: 'Your Font', sans-serif;
}
```
